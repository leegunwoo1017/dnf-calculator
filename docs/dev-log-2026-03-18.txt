================================================================================
[기술적 이슈 및 해결]
================================================================================

1. CORS 오류
   - 문제: 브라우저에서 네오플 API 직접 호출 시 CORS 차단
   - 해결: Vite의 server.proxy 설정으로 개발 서버에서 프록시 처리

2. 모험단 검색 제한
   - 문제: 네오플 API는 모험단명으로 직접 검색하는 기능 미지원
   - 현재 방식: 캐릭터명 검색 후 상세 정보에서 모험단명 조회
   - 향후 계획: 자체 DB 구축하여 모험단명 검색 기능 구현 예정

3. API 응답 구조
   - 캐릭터 검색 (/servers/{서버}/characters): 기본 정보만 반환 (모험단명 미포함)
   - 캐릭터 상세 (/servers/{서버}/characters/{캐릭터ID}): 모험단명 포함
   - 따라서 모험단 정보 표시를 위해 추가 API 호출 필요

[네오플 API 정보]
================================================================================

- API Gateway: https://api.neople.co.kr
- 인증: apikey 쿼리 파라미터 또는 헤더
- 요청 제한: 초당 1,000회
- 주요 엔드포인트:
  - 캐릭터 검색: /df/servers/{서버}/characters?characterName={이름}
  - 캐릭터 상세: /df/servers/{서버}/characters/{캐릭터ID}
- 서버 ID: cain, diregie, siroco, prey, casillas, hilder, anton, bakal, all
